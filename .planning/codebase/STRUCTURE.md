## Directory Purposes

**app:**
- Purpose: Contains all application source code and tests
- Contains: Java source, test classes, resource files, build configuration
- Key files: `build.gradle` (defines dependencies, native-image config, version management)

**app/src/main/java/com/anentropic/chuckd/:**
- Purpose: All production source code
- Contains: CLI command class, compatibility reporter, schema incompatibility model, version provider
- Key files:
  - `ChuckD.java` — Main command class and entry point
  - `CompatibilityReporter.java` — Schema comparison logic
  - `SchemaIncompatibility.java` — Result data model
  - `VersionProvider.java` — Version management

**app/src/main/resources/:**
- Purpose: Non-Java assets required at runtime
- Contains: GraalVM reflection configuration, version metadata
- Key files:
  - `reflection.json` — GraalVM native-image reflection config for schema providers
  - `version.properties` — Auto-generated version string

**app/src/test/java/com/anentropic/chuckd/:**
- Purpose: Unit test suite
- Contains: Test base class, format-specific test classes, compatibility reporter test
- Key files:
  - `ChuckDTestBase.java` — Shared test infrastructure
  - `ChuckDTestJSONSchema.java` — JSON Schema compatibility tests
  - `ChuckDTestAvro.java` — Avro compatibility tests
  - `ChuckDTestProtobuf.java` — Protobuf compatibility tests
  - `CompatibilityReporterTest.java` — Reporter logic tests

**app/src/test/resources/:**
- Purpose: Test fixtures — schema files for validation
- Contains: Three subdirectories with sample schemas
- Key files:
  - `jsonschema/*.json` — JSON Schema test files (person-base.json, person-narrowed.json, person-widened.json)
  - `avro/*.avsc` — Avro schema files
  - `protobuf/*.proto` — Protocol Buffer files

**bat-tests/:**
- Purpose: Integration tests for native image binary
- Contains: BATS shell test suite
- Key files: `smoke.bats` — Smoke tests that verify binary functionality

**gradle/:**
- Purpose: Gradle wrapper distribution
- Contains: Gradle distribution and wrapper configuration
- Generated by: Gradle wrapper initialization

**.github/:**
- Purpose: CI/CD automation
- Contains: GitHub Actions workflows for building, testing, releasing
- Key files: Workflow YAML files

**utils/:**
- Purpose: Helper scripts and Node.js dependencies (not core to application)
- Contains: npm modules for schema validation utilities
- Note: Appears to be separate tooling, not part of main build

**.planning/:**
- Purpose: GSD analysis and planning documents
- Contains: Architecture, structure, conventions, testing, and concerns documentation

## Naming Conventions

**Files:**
- Java source: `ClassName.java` using PascalCase (e.g., `ChuckD.java`, `CompatibilityReporter.java`)
- Test files: `*Test.java` or `*Tests.java` (e.g., `ChuckDTestJSONSchema.java`)
- Resource files: lowercase with hyphens (e.g., `person-base.json`, `reflection.json`)
- Build output: `.jar`, `.zip` artifacts with name from `archiveBaseName` setting

**Directories:**
- Java packages: reverse domain notation `com.anentropic.chuckd`
- Test resources: organized by schema format (`jsonschema/`, `avro/`, `protobuf/`)
- Gradle tasks: camelCase (e.g., `nativeCompile`, `shadowJar`)

**Classes:**
- Public classes: PascalCase (e.g., `ChuckD`, `CompatibilityReporter`)
- Enums: UPPERCASE (e.g., `SchemaFormat`, `CompatibilityLevel`, `LogLevel`, `OutputFormat`)
- Records: PascalCase (e.g., `SchemaIncompatibility`)

**Methods:**
- camelCase (e.g., `getStructuredReport()`, `compareJsonSchemas()`, `loadSchemas()`)
- Prefix `check` for validation: `check()` in CompatibilityReporter
- Prefix `format` for output: `formatText()`, `formatJson()`
- Prefix `get` for accessors: `getReport()` in test base
- Prefix `configure` for setup: `configureRootLogger()`
