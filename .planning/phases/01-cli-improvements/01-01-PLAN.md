    - --output JSON with incompatible schemas outputs issue array
    - --output TEXT with compatible schemas outputs nothing to stdout
    - --help footer shows exit code documentation
  </behavior>
  <action>
    Refactor ChuckD.java with these changes (order matters for coherence):

    1. **@Command annotation**: Add `exitCodeOnInvalidInput = 2`, `exitCodeOnExecutionException = 3`, and `footer` with exit code documentation table (see RESEARCH.md Pattern 2).

    2. **Replace dual @Parameters with single list**: Remove `File newSchemaFile` and `File[] previousSchemaFiles`. Add:
       ```java
       @Parameters(arity = "1..*", description = "Glob mode (1 arg)...\\nExplicit mode (2+ args)...")
       List<String> schemaArgs;
       ```

    3. **Add --quiet option**:
       ```java
       @Option(names = {"-q", "--quiet"}, description = "Suppress file metadata output on stderr")
       boolean quiet;
       ```

    4. **Add natural sort comparator** as a static inner class or private static method. Compare strings by splitting on digit/non-digit boundaries: digit chunks compared as Long.parseLong values, non-digit chunks compared lexicographically. Use `Long.parseLong` (not Integer) to avoid overflow on large numeric chunks.

    5. **Add expandGlob(String pattern) method**: Use `Path.of(pattern)` to split parent (search root) from filename. If parent is null, use `Path.of(".")`. Create `PathMatcher` with `"glob:" + filename`. Use `Files.walk(searchRoot, 1)` to list direct children (depth 1). Filter non-directories, match against `p.getFileName()`, sort with natural sort comparator. Return `List<Path>`.

    6. **Refactor loadSchemas()**: Change signature to accept `Path newSchemaPath` and `List<Path> previousPaths` instead of using fields directly. Load schemas from Path objects (convert via `.toFile()` for the existing `loadSchema` helper).

    7. **Refactor call()**: Implement dual-mode dispatch:
       - If `schemaArgs.size() == 1`: call `expandGlob()`. Handle 0-match (stderr error + return 2), 1-match (stderr info if !quiet + return 0). For 2+ matches, use result as sorted paths.
       - If `schemaArgs.size() >= 2`: convert to `List<Path>` directly via `Path.of()`.
       - In both modes: last path = newSchemaPath, all prior = previousPaths.
       - Print stderr metadata (previous files, new file) if `!quiet`.
       - Call refactored `loadSchemas(newSchemaPath, previousPaths)`.
       - Call `reporter.check(...)`.
       - JSON mode: always call `formatJson(issues)` and print (produces `[]` when empty).
       - TEXT mode: only print if `!issues.isEmpty()`.
       - Return `issues.isEmpty() ? 0 : 1`.

    8. **Refactor main()**: Add `IExitCodeExceptionMapper` to the CommandLine builder that maps `ParameterException` to 2 and everything else to 3. While `exitCodeOnInvalidInput = 2` handles parse errors, the mapper provides defense-in-depth. The mapper also ensures `exitCodeOnExecutionException = 3` is correctly applied for runtime exceptions thrown from `call()`.

    IMPORTANT: Do NOT call `System.exit()` inside `call()`. Return typed integers and let `CommandLine.execute()` handle it (existing pattern in main()).

    IMPORTANT: The `loadSchema()` helper takes a `File` param. Keep it as-is and call `.toFile()` on Path objects when passing to it. Or refactor to take Path if cleaner.

    IMPORTANT: `getStructuredReport()` is called by unit tests directly. It needs to work with the new field structure. Either make it accept parameters or have it read from the refactored fields. The simplest approach: keep `getStructuredReport()` working by having `call()` set internal fields (newSchema, previousSchemas) that `getStructuredReport()` still uses, OR refactor `getStructuredReport()` to accept paths. Choose whichever is cleaner -- this is Claude's discretion per CONTEXT.md.
