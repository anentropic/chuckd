Create /Users/paul/Documents/Dev/Personal/chuckd-action/src/validate.js with three exported pure functions:

1. `readInputs()` — reads all validation-related inputs via `core.getInput()`:
   - schemaPattern ('schema-pattern'), schemaFile ('schema-file'), previousSchemas ('previous-schemas')
   - baseRef ('base-ref'), format ('format'), compatibility ('compatibility')
   - logLevel ('log-level'), outputFormat ('output-format')
   - Returns plain object with camelCase keys

2. `detectMode(inputs)` — determines which mode to use:
   - Check emptiness using `!!value.trim()` (core.getInput returns '' for unset)
   - If schemaPattern set AND schemaFile/previousSchemas/baseRef all empty: return 'glob'
   - If schemaFile + previousSchemas both set AND schemaPattern/baseRef empty: return 'explicit'
   - If schemaFile + baseRef both set AND schemaPattern/previousSchemas empty: return 'git-compare'
   - Otherwise: throw Error with message: 'Invalid input combination. Use one of: (1) schema-pattern for glob mode, (2) schema-file + previous-schemas for explicit mode, (3) schema-file + base-ref for git-compare mode.'

3. `buildArgs(mode, inputs)` — builds chuckd CLI args array:
   - Start with empty array
   - If inputs.format is non-empty: push '--format', inputs.format
   - If inputs.compatibility is non-empty: push '--compatibility', inputs.compatibility
   - If inputs.logLevel is non-empty: push '--log-level', inputs.logLevel
   - If inputs.outputFormat is non-empty: push '--output', inputs.outputFormat
   - If mode === 'glob': push inputs.schemaPattern
   - If mode === 'explicit': split inputs.previousSchemas on /\s+/, filter(Boolean), push each, then push inputs.schemaFile
   - Return the args array

Import `@actions/core` for readInputs only. detectMode and buildArgs are pure functions (no imports needed in their bodies).
